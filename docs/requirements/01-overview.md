# 要件定義書（01-overview）
Auto Video AI Agent

本ドキュメントは、Auto Video AI Agent の全体要件を定義し、以降の設計・API仕様・アーキテクチャ・実装の基盤となる。

---

# 1. プロジェクト概要

Auto Video AI Agent は、**ユーザーが撮影した動画をアップロードするだけで、ショート動画向けの編集を完全自動化するAIエージェント**である。

編集という高負荷作業をゼロにし、社会人クリエイターが「撮るだけ」で継続的にSNS発信できる状態を実現する。

---

# 2. 開発背景とコンセプト

多くの社会人クリエイターは
- 編集に1〜2時間かかる
- 撮影後に意志力が尽きる
- 編集の学習コストが高い
という理由で継続が困難。

そこで本プロダクトは **圧倒的な“負担ゼロ”を目指す**。

> **動画アップ → few minutes → 完成動画DL**
→ 言い換えると “編集という概念を消す” のがゴール。

# 4. 解決する課題

### ❗ K1：編集作業が重く、継続できない
- 動画読み込み
- カット
- テロップ
- BGM
- サムネ
→ 全て意志力を消耗し、夜にはできない

### ❗ K2：毎回ゼロから構成を考える必要がある
- Hook → Body → CTA の構造化が負担

### ❗ K3：仕事後は脳疲労で作業が不可能
- 「編集ありき」の運用では永続不可能

### ❗ K4：動画クオリティがブレる
- 喋りのスムーズさ
- 冗長部分
- ロジック崩壊
→ 補正が必要

### ❗ K5：既存ツールは“補助止まり”で完全自動ではない
- Vrew, CapCut, Descript, Sora
→ 最終的に手動調整が必要で、本質的な負担は減らない

---

# 5. プロダクトのゴール（最重要）

### 🎯 **ゴール**
> **「撮るだけで動画が完成する」「撮影レベルの継続でSNS運用が成立する」**

### ⭐ 成功指標（KPI）
- 動画編集時間が **平均 40〜120分 → 3〜5分** へ
- 平均投稿数が **週1 → 週3以上** に増加
- 初期ユーザーの継続率 **70%超**
- アプリを使った投稿でフォロワー増加率が向上

---

# 6. MVP（最小プロダクト）の要件

**この範囲で十分価値が出る最小構成。**

### ◎ 必須
1. 動画アップロード（iPhone前提）
2. Whisper による文字起こし
3. 自動カット（無音区間・噛み・言い直しの除去）
4. 自動テロップ生成（字幕焼き込み or overlay）
5. 自動BGM
6. 自動サムネ抽出（自動で1枚）
7. 自動タイトル生成（AI）
8. 完成動画のダウンロード（mp4）

### ◎ MVPでやらない（後回し）
- 複雑な編集テンプレ
- 色味調整
- カット微調整UI
- セグメント編集
- ユーザーアカウント機能
- SNS連携
- AI台本生成（Phase2へ）

---

# 7. 将来的な拡張案（ロードマップ）

## Phase1：MVP

# 8. 非機能要件

| 項目 | 内容 |
|------|------|
| パフォーマンス | 1分動画の処理が2〜5分以内 |
| スケーラビリティ | 並列処理に耐えられる構成（キュー実装検討） |
| セキュリティ | 認証はPhase2以降、初期は限定公開 |
| デプロイ | Vercel + Supabase / Render / Railway を想定 |
| 可用性 | MVPは障害に強くなくてもよいが、復旧容易であること |

---

# 9. ユースケース（簡易）

### UC-01：動画をアップして完成動画を生成したい
- ユーザー：iPhoneで撮影
- アクション：アップロード
- システム：文字起こし→カット→テロップ→BGM→サムネ→出力
- 結果：ダウンロード可能

### UC-02：タイトル案が欲しい
- Whisper文字起こしから Claude/GPT が要点抽出
- タイトル/概要を数種類生成

---

# 10. 依存ライブラリ（想定）
- Whisper（音声認識）
- moviepy or ffmpeg-python（編集）
- OpenAI API / Claude API（文章補正・タイトル）
- FastAPI / Next.js API Routes
- TailwindCSS（UI）
- Supabase Storage（動画保存）

---

# 11. リスクと対策
| リスク | 対策 |
|--------|------|
| 処理時間が長い | 並列処理 / 事前圧縮 / バッチ化 |
| エラー復旧が難しい | ログ追跡 / retriable構造 |
| 音声認識精度不足 | モデル選択 / ノイズ除去 |
